1. Sơ đồ Class Tổng Thể (Class Diagram)

classDiagram
    %% --- Models ---
    class Poi {
        +int Id
        +string Name
        +string DisplayName
        +string DisplayTtsScript
        +double Latitude
        +double Longitude
        +double Radius
        +int Priority
        +string ImageUrl
        +GetSampleData() List_Poi
    }
    class FoodPlace {
        +string Id
        +string Name
        +string Description
        +string Address
        +double Rating
        +int ReviewCount
        +string ImageUrl
        +string NarrationText
    }
    class UserAccount {
        +int Id
        +string FullName
        +string Email
        +string Password
    }
    class FoodItem {
        +string Name
        +string Price
        +string ImageUrl
    }

    %% --- Services ---
    class PoiRepository {
        -SQLiteAsyncConnection _database
        -bool _isInitialized
        +InitAsync() Task
        +GetAllPoisAsync() Task_List_Poi
        +GetPoiAsync(int) Task_Poi
    }
    class DatabaseService {
        -SQLiteAsyncConnection _db
        +IsEmailExistsAsync(string) Task_bool
        +RegisterUserAsync(UserAccount) Task_int
        +GetUserAsync(string, string) Task_UserAccount
    }
    class NarrationEngine {
        +NarrationEngine Instance$
        +PlayNarrationAsync(FoodPlace, bool) Task
        +CancelCurrentNarration()
    }
    class MenuDataStore {
        +GetMenuForPlace(string)$
    }
    class LocalizationResourceManager {
        +string CurrentLanguageCode
        +LocalizationResourceManager Instance$
    }

    class HomeViewModel {
        +ObservableCollection_FoodPlace FeaturedPlaces
        -List_FoodPlace _allPlaces
        -PoiRepository _poiRepository
        +LoadDataAsync() Task
        +FilterByCategory(string)
    }

    class HomePage
    class PlaceDetailPage
    class MenuPage
    class MapPage
    class AudioPlayerPage
    class LoginPage
    class RegisterPage
    class SettingsPage

    HomePage --> HomeViewModel : binds
    HomeViewModel --> PoiRepository : reads
    PoiRepository --> Poi : return
    LoginPage --> DatabaseService : auth
    RegisterPage --> DatabaseService : creation
    DatabaseService --> UserAccount : model
    PlaceDetailPage --> MenuPage : navigates
    PlaceDetailPage --> AudioPlayerPage : navigates
    MenuPage --> MenuDataStore : reads
    MenuDataStore --> FoodItem : return
    AudioPlayerPage --> NarrationEngine : invokes
    MapPage --> NarrationEngine : invokes
    HomePage ..> LocalizationResourceManager : observes
    SettingsPage --> LocalizationResourceManager : updates

3. Lồng Hoạt Động Của Các Chức Năng Cốt Lõi (Activity + Sequence)

3.1. Chức năng Đăng Ký (Registration)

Activity Diagram - Lộ trình kiểm định đăng ký
flowchart TD
  A(Mở trang đăng ký) --> B[Nhập Họ tên Email Mật khẩu]
  B --> C[Nhấn nút Đăng ký]
  C --> D{Kiểm tra tính hợp lệ dữ liệu}
  D -- Dữ liệu trống --> E[Hiển thị thông báo nhập thiếu]
  E --> B
  D -- Hợp lệ --> F[Gọi hàm IsEmailExistsAsync]
  F --> G{Email trùng lặp?}
  G -- Trùng lặp --> H[Thông báo Email đã tồn tại]
  H --> B
  G -- Chưa tồn tại --> I[Gọi hàm RegisterUserAsync]
  I --> J[Tạo UserAccount và lưu vào SQLite]
  J --> K[Thông báo tạo tài khoản thành công]
  K --> L(Hoàn tất)
Sequence Diagram - Sự kiện truy vấn đăng ký
sequenceDiagram
    actor U as Người dùng
    participant P as RegisterPage
    participant DB as DatabaseService
    
    U->>P: Nhập thông tin và nhấn Đăng ký
    P->>DB: IsEmailExistsAsync
    activate DB
    DB-->>P: Kết quả true hoặc false
    deactivate DB

    alt Email đã tồn tại
        P-->>U: Hiển thị lỗi tài khoản đã tồn tại
    else Email hợp lệ
        P->>DB: RegisterUserAsync
        activate DB
        DB->>DB: Cập nhật CSDL SQLite
        DB-->>P: Trả về ID người dùng mới
        deactivate DB
        P-->>U: Hiển thị thông báo thành công và chuyển trang
    end

3.6. Chức năng Audio Narration (Text To Speech OS)

Activity Diagram - Logic chặn chống Spam quá tải Audio OS
flowchart TD
  A(Yêu cầu phát Audio) --> B[Lệnh Singleton NarrationEngine xử lý]
  B --> C[Kiểm tra thời gian Cooldown System]
  C --> D{Vi phạm yêu cầu?}
  D -- Có vi phạm --> E[Hủy tiến trình hiện tại tránh chồng lấp]
  D -- Hợp lệ --> F[Gọi CancelCurrentNarration dọn luồng cũ]
  F --> G[Cấu hình tham số Pitch và Locale mặc định OS]
  G --> H[Khởi chạy tác vụ nền TextToSpeech tĩnh SpeakAsync]
  H --> I{Ngoại lệ trong vòng Thread?}
  I -- Bị ép tắt thủ công --> J[Kích hoạt CancellationToken chặn họng]
  I -- Không phát sinh lỗi --> K[Hoàn thành chuẩn trạng thái báo Success]
  J --> L(Throw C# TaskCanceled khôi phục giao diện UI)
  K --> L
Sequence Diagram - Quản lý Thread Hệ thống đa luồng Engine Narrator
sequenceDiagram
    actor U as Người dùng
    participant UI as Giao diện Player Base
    participant NE as NarrationEngine Global
    participant TTS as Hạ Tầng Native OS Core (TextToSpeech API)
    
    U->>UI: Input tín hiệu Button Start Media
    UI->>NE: Truyền Callback PlayNarrationAsync chứa Document
    activate NE
    NE->>NE: Thuật toán kiểm duyệt ranh giới trễ Thread Anti Spam
    NE->>NE: Chấm dứt tác vụ nền API nếu vẫn còn tác vụ phát sinh trước đó
    NE->>TTS: Inject cấu hình Configuration Locale Code vào Class
    NE->>TTS: Khởi sinh hàm Native SpeakAsync khóa chung CancellationToken Source
    activate TTS
    
    alt Tác động ngoại vi chủ động cắt luồng
        U->>UI: Bật cờ BackStack PopAsync ngắt chu trình xem trang
        UI->>NE: Method thu hồi Event dừng khẩn
        NE->>TTS: Kích hoạt tham chiếu cờ HỦY của Cancellation Tác vụ nguồn
        TTS-->>NE: Chặn đứng thiết bị âm thanh Native đẩy ngược Catch Try
    else Xử lý vòng lặp API bình ổn đến cuối Buffer
        TTS-->>NE: Đánh dấu Completed Response báo cáo cỗ máy an toàn dọn Task
    end
    deactivate TTS
    
    NE-->>UI: Invoke Delegate lệnh Rendering thiết lập Button Active UI
    deactivate NE
